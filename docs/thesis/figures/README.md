# Thesis figures

## The three pipeline/architecture diagrams (two candidate styles)

Each figure carries the thesis's structural claim: **across all phases, only the
likelihood changes; the graph-GP prior is reused unchanged.** Corollaries encoded:
Phase 1's visible/hidden split is an evaluation device, not part of the model;
Phase 3 has no visible/hidden split at all (everything latent, conjugacy lost);
the model is two branches that fuse — graph *structure* and *evidence* (side
information).

| Style | Files | Character |
|---|---|---|
| **A — restrained schematic** | `pipeline_phase{1,2,3}.{tex,pdf,svg}` | Thin rules, near-monochrome, one accent. Dashed enclosure = shared prior. |
| **B — architecture-diagram** | `arch_phase{1,2,3}.{tex,pdf,svg}` | Isometric volumes, Okabe–Ito colour, two colour-coded branches, circled operators, glyph outputs. |

**The choice between A and B is Ray's; do not consolidate them unilaterally.**
Neither style is referenced from `draft.tex` yet. Suggested placement once chosen:
Phase 1 in the model chapter beside the joint-kernel equation; Phases 2 and 3 in
Future Work.

## Style B design language (do not shuffle)

Palette: Okabe–Ito, colourblind-safe. Semantic assignment:

| Colour | Meaning |
|---|---|
| blue `#0072B2` | graph-structure branch: score graph → `L_G` → `g(ν;s)` → `K_G` |
| amber `#E69F00` | evidence branch: score features `X_feat`; music model → `H`; linear kernels |
| green `#009E73` | fusion: `⊕` summing Kronecker terms + noise → joint kernel `K` |
| pink `#CC79A7` | inference head |
| vermillion `#D55E00` | **the likelihood — the only element that changes across phases** |

Isometric cuboids = data volumes (shape annotations underneath are what make the
figure teach); rounded rectangles = operations; offset stacks = repetition;
**dashed stroke = not yet implemented** (never dash something that exists).
Vocabulary: figures follow the thesis register — "the proposed model", "music
model", "two-stage plug-in baseline"; no project-internal codenames. Nothing may
appear in a figure that the thesis does not define.

Dimensions are real, not invented: `N×26` = 25 rich score features + bias
(`baselines.rich_score_features` + bias column, as assembled in
`scripts/eval_graphgp.py`); `N×512` = the Phase-0 checkpoint's `d_model`.
Phase-2 `k×k` stays symbolic until the vibrato parameterisation is fixed.

## Build (single source of truth = the .tex files)

```bash
cd docs/thesis/figures
tectonic <file>.tex                    # or pdflatex
pdftoppm -png -r 105 -singlefile <file>.pdf /tmp/preview   # then LOOK at it
pdftocairo -svg <file>.pdf <file>.svg  # SVGs are DERIVED from the PDFs
```

The `.svg` files are generated with `pdftocairo` — do not hand-edit them; edit the
`.tex` and regenerate both PDF and SVG. These figures can compile cleanly while
being visually broken (overlaps, in-box hyphenation, floating volumes), so the
render-and-look loop is not optional; box/annotation styles set
`\hyphenpenalty=10000` to keep words from hyphenating inside boxes. Figure PDFs
are tracked deliberately (`.gitignore` negation) because `\includegraphics` needs
them on a fresh clone.
